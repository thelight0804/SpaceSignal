# 1. Project Scope & Identity (SpaceSignal)

**Status:** Accepted
**Date:** 2025-12-14
**Last Updated:** 2025-12-15

## Context (배경 및 문제점)
- 단순한 웹 서비스 개발을 넘어, 위성 도메인 지식과 실시간 데이터 처리 역량을 결합한 프로젝트가 필요함.
- 초기 아이디어였던 "위성 통신 내용 분석"은 법적/기술적 리스크가 있어, 안전하면서도 기술적 깊이를 보여줄 수 있는 대안이 필요함.

## Decision (결정 사항)
프로젝트명을 **SpaceSignal**로 정하고, 다음과 같이 범위를 정의한다.

1. **Core:** 전 세계 위성의 실시간 위치 추적(TLE 기반) 및 시각화.
2. **Expansion:** 통신 로그뿐만 아니라, **위성이 송신하는 신호 정보(Telemetry/Signal Metadata)**를 수집하고 시각화하여 기술적 깊이를 더함.
3. **Target:** 전 세계를 대상으로 하되, **국가/지역별 지상국 필터링(Region Filtering)** 기능을 도입하여 데이터 탐색의 편의성을 높임.

## Consequences (결과 및 효과)
### Positive (장점)
- **도메인 심화:** 단순 위치 추적을 넘어 신호/주파수 데이터를 다룸으로써 실제 위성 운용 환경과 유사한 경험을 쌓음.
- **안전성:** 공개된 TLE 및 관측 로그만 사용하여 법적 리스크 제거.
- **확장성:** 위치 추적(Phase 1) → 신호/로그 분석(Phase 2) → 지상국 관리(Phase 3)로 단계적 확장이 용이함.

### Negative (단점/리스크)
- **차별화:** 단순 위성 추적 서비스는 이미 많으므로, '신호 정보' 시각화 기능 구현이 프로젝트의 핵심 차별화 포인트가 됨.
