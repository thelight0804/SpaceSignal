# 2025-12-23-다국어 README 추가 및 문서 표준화

## Summary (회의 요약)
README.md를 ADR에 맞춰 업데이트하고, 영어 및 일본어 버전 README를 추가함. 파일명 규칙을 검토하여 표준 형식으로 변경함.

## Key Decisions (주요 결정 사항)

### README.md Tech Stack 섹션 업데이트
- **내용:** Infrastructure에 Terraform/GitHub Actions 추가. Database를 "PostgreSQL (Planned)"에서 "Amazon RDS for PostgreSQL 14+"로 구체화.
- **이유:** ADR-004(Database), ADR-005(Terraform), ADR-006(CI/CD)의 결정사항을 README에 명확히 반영하여 문서 일관성을 유지하기 위함.

### 영어 & 일본어 README 추가 및 파일명 표준화
- **내용:** 영어(README.en.md) 및 일본어(README.ja.md) 버전 README를 추가하고, 한국어는 기본 README.md로 유지함.
- **이유:** IETF 언어 태그 표준(BCP 47)에 따라 소문자 언어 코드 사용. 주요 오픈소스 프로젝트(Vue.js, React, Kubernetes)의 관행을 따라 `.`(점) 구분자를 사용하여 파일 확장자 형식으로 표기함. 일관된 네이밍 규칙(README.{lang}.md)을 적용하여 다국어 지원 문서의 일관성을 유지함.

### 다국어 README 간 언어 전환 링크 추가
- **내용:** 각 README 파일(README.md, README.en.md, README.ja.md) 상단에 다른 언어 버전으로 이동할 수 있는 하이퍼링크를 추가함. 형식: `[English](README.en.md) | [日本語](README.ja.md)`
- **이유:** GitHub에서 README를 읽는 사용자가 쉽게 선호하는 언어로 전환할 수 있도록 UX 개선. Vue.js, React, Kubernetes 등 주요 글로벌 오픈소스 프로젝트에서 널리 사용되는 방식을 따름.

## TIL (느낀 점)
- 다국어 문서 지원 시 일관된 네이밍 규칙(README.{lang}.md)을 유지하면 프로젝트의 전문성과 접근성이 높아짐.
- README 파일 간 언어 전환 링크를 상단에 배치하는 것이 주요 글로벌 오픈소스 프로젝트에서 널리 사용되는 일반적인 패턴임을 확인함.
