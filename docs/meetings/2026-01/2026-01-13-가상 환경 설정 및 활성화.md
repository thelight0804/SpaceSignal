# 2026-01-13-가상 환경 설정 및 활성화

## Summary (회의 요약)
Python 가상 환경 생성 방법을 학습하고, `.venv` 가상 환경을 사용하기로 결정함. FastAPI 개발을 위한 로컬 Python 환경 준비 완료.

## Key Decisions (주요 결정 사항)

### 가상 환경 사용 결정
- **내용:** 프로젝트에 `.venv` 가상 환경을 활성화하여 사용.
- **이유:** 
  - `.venv`는 Python 가상 환경의 표준 명명 관례를 따름.

### 가상 환경 활성화 방법
- **내용:** CMD에서 `.venv\Scripts\activate.bat` 실행.

## Blockers & Solutions (블로커 & 해결책)
- **문제:** 가상 환경 생성 명령어 `python -m venv .spaceSiganl`에 대한 혼란. 새로 만들어야 하는지, 어떤 이름을 사용해야 하는지 불명확했음.
- **해결:** 프로젝트 디렉토리 구조를 확인하여 `.venv/`가 이미 존재함을 발견. 새로 만들지 않고 기존 가상 환경을 활성화하여 사용하기로 결정.

## TIL (느낀 점)
- `python -m venv <디렉토리명>` 명령어로 Python 가상 환경을 생성할 수 있음.
- 일반적으로 `.venv` 이름을 사용하는 것이 Python 커뮤니티의 관례.
